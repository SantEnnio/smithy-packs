# CS0103 — Undefined identifier

Meaning: a name is referenced but is not visible in the current scope.

Unity-aware checks, in order:

1. **Is it a field that should appear in the Inspector?** Declare
   `[SerializeField] private <Type> <name>;` at the top of the class.
2. **Is it another component?** Either cache it via `GetComponent<T>()`
   in `Awake`, or expose it as a serialized reference and set it in the
   Inspector.
3. **Is it a typo?** Patch only the misspelled identifier — do NOT rewrite
   the surrounding code.
4. **Is it a Unity API the model wants to invent?** Do not invent APIs.
   Read the existing code first.

Repair recipe:

1. Read ±15 lines around the diagnostic line with `read_file`.
2. Search the same file for a similarly-named identifier (rename candidate).
3. Apply the minimal `patch`.
4. Recompile.
