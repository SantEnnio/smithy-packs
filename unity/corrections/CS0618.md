# CS0618 — Obsolete API (Unity 6 renames)

Meaning: the member still compiles but is DEPRECATED. It is only a warning, so
the build passes — but you MUST modernise it. Do not leave obsolete API in place,
and never add new obsolete calls.

Unity 6 (6000.x) renamed several common APIs. Apply the replacement:

- `Rigidbody.drag`            → `Rigidbody.linearDamping`
- `Rigidbody.angularDrag`     → `Rigidbody.angularDamping`
- `Rigidbody.velocity`        → `Rigidbody.linearVelocity`
- `Rigidbody2D.velocity`      → `Rigidbody2D.linearVelocity`
- `Object.FindObjectOfType<T>()`     → `Object.FindFirstObjectByType<T>()`
- `Object.FindObjectsOfType<T>()`    → `Object.FindObjectsByType<T>(FindObjectsSortMode.None)`

The message itself names the replacement (`(UnityUpgradable) -> linearDamping`).

Repair recipe:

1. For EVERY CS0618 diagnostic, read the named line.
2. Replace the obsolete member with the modern one above (the message's
   `-> <name>` tells you which).
3. Apply a minimal `patch` per file — do not rewrite whole files.
4. Recompile; there must be ZERO CS0618 warnings left.
