# CS0029 — Cannot implicitly convert type

A value of type `A` is being assigned to a slot of type `B` and the
compiler does not know how to convert.  Common Unity cases:

- `Vector2` ↔ `Vector3`: use `new Vector3(v.x, v.y, 0)` or `(Vector2)v3`.
- `int` ↔ `float`: explicit `(float)i` or `(int)f`.
- Generic `List<T>` ↔ `T[]`: call `.ToArray()` / `.ToList()`.
- `Transform` ↔ `GameObject`: use `.transform` or `.gameObject`.

## Repair recipe

1. Read ±10 lines around the diagnostic.
2. Apply the **smallest** explicit conversion that makes the types match.
3. Do not introduce a new variable when a cast suffices.
4. Recompile.
