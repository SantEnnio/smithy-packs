# CS0246 — Type or namespace not found

Meaning: the compiler cannot resolve the named type.

Most common causes in Unity:

1. **Missing `using` directive** — add `using UnityEngine;` /
   `using System.Collections.Generic;` / `using TMPro;` at the top.
2. **Missing assembly reference** — the type lives in another assembly
   that this `.asmdef` does not reference.  Add the reference in the
   `.asmdef` file.
3. **Misspelled type name** — e.g. `UnityEngne` instead of `UnityEngine`.

Repair recipe:

1. Read the file head to see which `using`s are already present.
2. If a `using` is missing, add it as a single `patch` at the top.
3. If the type is in another assembly, update the `.asmdef`.
